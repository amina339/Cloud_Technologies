## Ход выполнения работы 

### 1. Импорт и обработка 

Был произведен импорт файла данных биллинга (CSV) в MS Excel с использованием разделителя «точка с запятой». 

### 2. Классификация сервисов (Mapping)

На основе документации Microsoft Azure и принципов классификации, выработанных в Лабораторной работе №1 (AWS), было выполнено сопоставление сырых данных с унифицированной иерархией (IT Tower -> Service Family -> Service Type -> Service Sub Type).

При заполнении таблицы я руководствовалась следующей логикой:

1. Названия категорий (IT Tower, Service Family) были унифицированы с классификацией AWS из прошлой работы для формирования независимого от вендора видения.

2. Анализ Compute vs Storage: В рамках сервисов баз данных (SQL Database) было произведено разделение на вычислительные мощности (vCore, DTU отнесены к Instances) и хранение резервных копий (Backup Storage отнесено к Storage).

3. Выделение лицензий: В категории Virtual Machines строки, содержащие стоимость программного обеспечения (Red Hat, SQL Server, BizTalk), были выделены в подтип Software License, чтобы отделить их от стоимости инфраструктуры.

4. Обработка пустых категорий: В случаях, когда поле Meter Sub-Category было пустым, классификация производилась на основе детального описания в Meter Name (например, определение типа хранилища или лицензии).

### 3. Построение иерархии и проверка

Для выполнения требования по распределению потребления от «большего к меньшему» была сформирована 5-уровневая структура данных.

IT Tower (Глобальная категория, например, Compute).

Service Family (Семейство, например, Compute Instances).

Service Type (Тип сервиса, например, Virtual Machines).

Service Sub Type (Подтип использования, например, Software License).

Это позволило визуализировать структуру затрат и убедиться, что каждый сервис Azure корректно "проваливается" до конкретного типа использования (Usage Type).
