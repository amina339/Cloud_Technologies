## Вариант работы
Был взят .csv файл 7 варианта 
## Шаги работы
1. Файл был импортировал в excel с определением соотвествующего разделителя (точка с запятой)
2. По product code был написан service type. В образце по сути service type был названием самого  сервиса, поэтому по аналогии указаны были названия
3. По product code был написан IT Tower, который определял самое широкое описание-применение сервиса. Названия по большей части тоже взяты из образца
4. По product code + IT Tower был написан service family, определяющий основное предназначение сервиса (менее широкий чем IT Tower, конечно)
5. Далее пришлось полазить по документациям типа https://docs.aws.amazon.com/AmazonS3/latest/userguide/aws-usage-report-understand.html и соответствующим остальным сервисам

Там уже нужно было найти конректные Usage Type, Units, Granularity и Description и по ним написать Service Sub Type и Service Usage Type (за что платят и как платят)

В основном ориентир был на SQL-выклдаки в столбце UsageType. Они содержали основные ключевые слова и нужную информацию по конкретным типам 

Таким образом и заполнялись нужные строки: поиск в документации - сравнение - краткая выжимка 

## Вывод
В ходе работы была успешно построена иерархическая классификация облачных сервисов на основе данных биллинга, от общих категорий (IT Tower) до конкретных моделей потребления (Service Usage Type).

Мы научились сопоставлять технические данные биллинга провайдера (Product Code, Usage Type, LineItemDescription) с категориями сервисной модели.
